# Documentation

During the documentation step, the agent updates the
documentation. See [git.md](./git.md) for instructions about the use
of Git.

In the `doc-only` track, it follows the changes described in the task
description.

In the `code` track, it reflects in the documentation of the software
the changes that were made during the `execution` step.

In the `workflow` track, it updates the Markdown documents of this
repository and the other documentation files referencing the cue
workflow.

Once the documentation is fully updated, the agent transitions the
task to the `agent-review` status.
