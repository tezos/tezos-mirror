# lib_srs_extraction

Extracts SRS (Structured Reference Strings) for G1 and G2 from powers-of-tau
files generated by the ZCash and Filecoin MPC ceremonies:
- <https://download.z.cash/downloads/powersoftau/> (max 2^21)
- <https://trusted-setup.filecoin.io/phase1> (max 2^27)

## What is an SRS?

A Structured Reference String (SRS) is a sequence of elliptic curve points of
the form (g, g·τ, g·τ², …, g·τⁿ⁻¹), where τ is a secret scalar established by
the MPC ceremony. These points are the trusted setup for the KZG polynomial
commitment scheme used in the DAL (Data Availability Layer) cryptobox.

## Committed test files

The following binary SRS files are committed to the repository so that tests in
`src/lib_bls12_381_polynomial/test/`, `src/lib_crypto_dal/test/`, and
`src/lib_srs_extraction/test/` can run without downloading the large raw
ceremony files from the internet.

### Naming convention

Files follow the pattern `srs_{ceremony}_{group}_{power}`:

- `ceremony`: the MPC ceremony (`zcash` or `filecoin`)
- `group`: the BLS12-381 group (`g1` or `g2`)
- `power`: base-2 logarithm of the number of points (e.g. `5` means 2⁵ = 32 points)

### Files

| File                | Ceremony | Group | Points |
| ------------------- | -------- | ----- | ------ |
| `srs_zcash_g1_5`    | ZCash    | G1    | 32     |
| `srs_zcash_g2_5`    | ZCash    | G2    | 32     |
| `srs_filecoin_g1_6` | Filecoin | G1    | 64     |
| `srs_filecoin_g2_6` | Filecoin | G2    | 64     |

### Binary format

Each file is a flat sequence of compressed BLS12-381 group elements with no
header:

- G1 elements: 48 bytes each
- G2 elements: 96 bytes each

To truncate an existing file to fewer elements:

```bash
head -c $(( 48 * nb_elements )) srs_zcash_g1_5 > srs_zcash_g1_4   # G1
head -c $(( 96 * nb_elements )) srs_zcash_g2_5 > srs_zcash_g2_4   # G2
```

### Regenerating the files

The regression tests in `src/lib_srs_extraction/test/` verify that the committed
files are bit-for-bit identical to a fresh extraction from the raw ceremony file,
and also check the pairing equations on the result.

To regenerate:

1. Download the raw powers-of-tau file for the ceremony and power you need:
   - ZCash: <https://download.z.cash/downloads/powersoftau/phase1radix2m5>
   - Filecoin: `phase1radix2m6` from <https://trusted-setup.filecoin.io/phase1>

2. Build the extraction tool:

   ```bash
   dune build src/lib_srs_extraction/srs_extraction_main.exe
   ```

3. Run the extraction (from the directory containing the raw file):

   ```bash
   # ZCash
   path/to/srs_extraction_main.exe extract zcash g1 phase1radix2m5 -o srs_zcash_g1_5
   path/to/srs_extraction_main.exe extract zcash g2 phase1radix2m5 -o srs_zcash_g2_5

   # Filecoin
   path/to/srs_extraction_main.exe extract filecoin g1 phase1radix2m6 -o srs_filecoin_g1_6
   path/to/srs_extraction_main.exe extract filecoin g2 phase1radix2m6 -o srs_filecoin_g2_6
   ```

4. Copy the new files into every test directory that depends on them (see the
   `dep_files` entries in `manifest/product_octez.ml`).

5. Verify with the regression tests:

   ```bash
   dune runtest src/lib_srs_extraction/test/
   ```
